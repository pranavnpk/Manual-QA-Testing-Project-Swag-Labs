# Manual QA Testing Project – Swag Labs

## Project Overview
This project focuses on manual testing of the Swag Labs demo e-commerce application. The objective was to validate application functionality, identify defects, document test cases, and practice API testing using Postman.

## Application Tested
Swag Labs Demo E-Commerce Application

## Modules Tested
- Login Module
- Products Module
- Cart Module
- Checkout Module
- Menu & Logout Module

## Testing Types Performed
- Functional Testing
- Smoke Testing
- Regression Testing
- Exploratory Testing
- Edge Case Testing
- API Testing

## Test Case Documentation
Created and executed 60+ manual test cases covering:
- Positive scenarios
- Negative scenarios
- Edge cases
- UI validations
- Navigation testing

## Bug Reporting
Documented usability issues and validation observations including:
- UI alignment issues
- Cart badge validation observations
- Checkout validation handling
- Session handling observations

## API Testing
Performed API testing using Postman and FakeStoreAPI:
- GET Requests
- POST Requests
- Status Code Validation
- JSON Response Validation
- Invalid Endpoint Testing
- Negative Testing

- VS Code

## Key Learnings
- Manual Testing Process
- Test Case Design
- Bug Reporting Workflow
- API Testing Basics
- Frontend vs Backend Validation
- QA Documentation Practices

## Outcome
Successfully performed manual testing on a demo e-commerce application, created professional QA documentation, practiced API testing workflows, and gained hands-on understanding of software testing methodologies.
